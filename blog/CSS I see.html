BFC(Block Formatting Content)/格式化上下文

Describe BFC(Block Formatting Context) and how it works.

简单来说BFC会形成一个独立的容器，该容器不受外界相邻元素的影响，同时也不影响外界元素(可以避免margin穿透，清除浮动)。
触发BFC的方法有：
  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block, table-cell, table-caption, flex, inline-flex
  • overflow不为visible

布局规则：
  • 内部的Box会在垂直方向，一个接一个地放置
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  • 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算

tips/注意

diaplay:inline/block和标签的嵌套规则

对于display属性来书，要记住的一点是，这只是改变了元素的显示角色，而不是本质。换句话说设置一个行内元素display:block来生成块框并不会把这个行内元素变成一个真正的块级元素。所以，链接可以放在一个段落中，但是链接不能包含段落，无论如果设计元素的样式，都不行。
这里顺便提一下标签的嵌套规则：
  1. 块级元素可以包含内联元素或 某些块级元素，但内联元素不能包含块级元素，它只能包含其它内联元素。 块级元素
  2. 块级元素不能放在p里面。
  3. 有几个特殊的块级元素只能包含内联元素，不能包含块级元素。如h1,h2,h3,h4,h5,h6,p,dt
  4. li内可以包含div
  5. 块级元素与块级元素并列、内联元素与内联元素并列。（错误的：<div><h2></h2><span></span></div>)

display:inline-block/行内块元素

行内块元素作为替换元素放在行中。这说明，行内元素的底端默认地位于文本行的基线上，而且内部没有行分隔符。

clip/内容裁剪

clip可以改变剪裁区域的形状，在设置overflow为要剪裁该内容时候。它只是改变现实内容的区域形状。
值： rect(top,right,bottom,left)|auto|inherit
应用于：绝对定位元素(在CSS2中，clip应用于块级元素和替换元素))
要注意的是: rect(....)的值不是边偏移，而是距元素左上角的距离。

visibility/元素可见性

这个属性，首先要掌握的是:它和display:none的区别，后者不单单元素不显示，还会从文档中删除，其实在浏览器的构成 render Tree的时候，该元素就不存在了。而前者，只是不可以看见，还是在文档中，影响着布局。
除此之外，要注意：
对于 visibility:collapse,根据CSS2，如果用于非表元素，其和hidden一样。
对于一个 hidden的元素，其后代元素还是可以可见的，但是要在其后代元素上显示声明visible。当然这也说明，visibility属性是可以继承的！
