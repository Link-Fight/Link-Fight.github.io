BFC(Block Formatting Content)/格式化上下文

Describe BFC(Block Formatting Context) and how it works.

简单来说BFC会形成一个独立的容器，该容器不受外界相邻元素的影响，同时也不影响外界元素(可以避免margin穿透，清除浮动)。
触发BFC的方法有：
  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block, table-cell, table-caption, flex, inline-flex
  • overflow不为visible

布局规则：
  • 内部的Box会在垂直方向，一个接一个地放置
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  • 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算

  • 双飞翼布局使用了：浮动，负边距。
  • 圣杯布局使用了：浮动，负边距，相对定位。

  • 使用浮动，可以让块级元素可以布局在同一行。但是这样需要让中间(Main)的宽度为自适应(100%),会把后面添加的浮动元素挤到下一行。
  • 此时，使用负边距让浮动元素在水平方向上移动。 当负边距设置比较大的值时候，将从当前行移动到上一行。例如：这里的对 left设置： margin-left: -100%将把left移动到上一层Main元素的顶部。不过通过负边距把左右部分分别移动到Main的左右两边的时候，也会占据并遮挡左右两边的内容。
  • 而两者不同就在于如何解决左右两边被屏蔽的问题。双飞翼就采用在Main里添加元素div并设置左右margin值。 而圣杯布局就采用在 Main上设置左右 padding 值，并在左右内容设置相对定位。

自适应的正方形

使用纯 CSS 制作出能够自适应大小的正方形

除了CSS3 中的视口相对单位，主要利用到 margin, padding 的百分比数值相对父元素宽度的宽度计算得出 来制作宽高相等、且相对视口宽度自适应的正方形。如果需求是制作相对视口高度自适应的正方形，估计就只能使用 vh 单位了
